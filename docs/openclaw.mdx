OpenClaw is an AI coding assistant that communicates over the [Model Context Protocol](https://modelcontextprotocol.io) (MCP).
By connecting OpenSRE as an MCP server inside OpenClaw, you can run root-cause analysis and catch production errors without leaving your editor.

---

## How it works

There are **two directions** to this integration — pick the one that matches your workflow:

| Direction | What it does |
|---|---|
| **OpenClaw → OpenSRE** (this page) | OpenClaw calls OpenSRE's `run_rca` tool to investigate alerts from your IDE |
| **OpenSRE → OpenClaw** | OpenSRE resolves your OpenClaw MCP bridge as a credential and can call back into OpenClaw tools during an investigation |

Most users only need **direction 1**. Direction 2 is for advanced pipelines where OpenSRE orchestrates OpenClaw as a tool.

---

## Direction 1: OpenClaw calls OpenSRE (recommended)

### Step 1: Start the OpenSRE MCP server

Install OpenSRE and start the server:

```bash
pip install opensre
opensre-mcp
```

This starts an MCP server on `stdio` that exposes the `run_rca` tool.

### Step 2: Add OpenSRE to OpenClaw's MCP config

In OpenClaw, open **Settings → MCP Servers** (or edit your `mcp_config.json`) and add:

```json
{
  "mcpServers": {
    "opensre": {
      "command": "opensre-mcp",
      "args": []
    }
  }
}
```

<Tip>
If `opensre-mcp` is not on your PATH, use the full path: `"/path/to/venv/bin/opensre-mcp"`.
</Tip>

### Step 3: Set your integrations

OpenSRE needs at least one observability integration (Datadog, Grafana, Sentry, etc.) to investigate alerts.
Set environment variables before starting OpenClaw:

```bash
# Example: Datadog
export DD_API_KEY=your_api_key
export DD_APP_KEY=your_app_key

# Example: Sentry
export SENTRY_ORG_SLUG=my-org
export SENTRY_AUTH_TOKEN=sntrys_...
```

Or run the setup wizard once:

```bash
opensre integrations setup
```

This writes credentials to `~/.tracer/integrations.json` so they persist across sessions.

### Step 4: Investigate from OpenClaw

In OpenClaw, use the `run_rca` tool directly in chat:

```
Run an investigation on this alert:
{"title": "High error rate on /api/checkout", "service": "checkout", "severity": "critical"}
```

OpenSRE will query your connected integrations, correlate logs and metrics, and return a structured root-cause analysis.

A ready-to-paste example tool payload is available in the repo at:

```text
tests/fixtures/openclaw_test_alert.json
```

## Direction 2: OpenSRE resolves OpenClaw as an integration

This allows the OpenSRE investigation pipeline to call tools exposed by your OpenClaw MCP bridge.

### Option A: Environment variables

```bash
OPENCLAW_MCP_MODE=stdio
OPENCLAW_MCP_COMMAND=openclaw
OPENCLAW_MCP_ARGS="mcp serve"
```

| Variable | Default | Description |
|---|---|---|
| `OPENCLAW_MCP_MODE` | `stdio` in the wizard | Transport: `streamable-http`, `sse`, or `stdio` |
| `OPENCLAW_MCP_AUTH_TOKEN` | — | Bearer token for authentication |
| `OPENCLAW_MCP_COMMAND` | — | Executable to launch (stdio mode only) |
| `OPENCLAW_MCP_ARGS` | — | Space-separated args for stdio mode |
| `OPENCLAW_MCP_URL` | — | Custom OpenClaw MCP endpoint URL (HTTP/SSE modes only) |

For advanced/custom remote MCP transports only:

```bash
OPENCLAW_MCP_MODE=streamable-http
OPENCLAW_MCP_URL=https://your-custom-openclaw-mcp-endpoint
OPENCLAW_MCP_AUTH_TOKEN=your_bearer_token
```

### Option B: Interactive CLI

```bash
opensre integrations setup
```

Select **OpenClaw** when prompted.

### Option C: Persistent store (`~/.tracer/integrations.json`)

```json
{
  "version": 1,
  "integrations": [
    {
      "id": "openclaw-prod",
      "service": "openclaw",
      "status": "active",
      "credentials": {
        "mode": "stdio",
        "command": "openclaw",
        "args": ["mcp", "serve"]
      }
    }
  ]
}
```

### Verify the integration

```bash
opensre integrations verify --service openclaw
```

Expected output when connected:

```
  SERVICE    SOURCE       STATUS      DETAIL
  openclaw   local env    passed      OpenClaw MCP connected via stdio (openclaw); discovered 8 tool(s).
```

## Transport modes

| Mode | When to use |
|---|---|
| `stdio` | Recommended for OpenClaw itself. Use `openclaw mcp serve` |
| `streamable-http` | Advanced/custom remote MCP transport if you expose the bridge through HTTP streaming |
| `sse` | Advanced/custom remote MCP transport if you expose the bridge through SSE |

## Catching errors in OpenClaw

### Pattern 1: Paste an error, trigger investigation

Copy a stack trace or error message into OpenClaw and ask:

```
This error appeared in production 10 minutes ago. Investigate it:

TypeError: Cannot read properties of undefined (reading 'userId')
  at /app/checkout/handler.js:42:18
  ...
```

OpenClaw calls `run_rca` → OpenSRE queries your logs and returns a root-cause analysis in your editor.

### Pattern 2: Trigger from an alert webhook

Point your alerting tool (Datadog, Grafana, PagerDuty) at OpenSRE's webhook endpoint, then review findings in OpenClaw:

```bash
opensre serve  # starts the webhook server
```

When an alert fires, OpenSRE runs the investigation automatically. You can then ask OpenClaw to show you the latest findings:

```
Show me the most recent OpenSRE investigation result.
```

### Pattern 3: Investigate by service name

```
Run an OpenSRE investigation for the payment-service — it had a spike in 5xx errors at 14:30 UTC today.
```

## Troubleshooting

| Symptom | Fix |
|---|---|
| **`opensre-mcp` not found** | Ensure `opensre` is installed in the same Python environment OpenClaw uses, or provide the full path |
| **`run_rca` returns "No integrations configured"** | Set at least one observability env var (e.g. `DD_API_KEY` + `DD_APP_KEY`) or run `opensre integrations setup` |
| **OpenClaw verify fails against `http://127.0.0.1:18789/`** | That URL is OpenClaw's Control UI/Gateway, not the MCP bridge. Use `OPENCLAW_MCP_MODE=stdio`, `OPENCLAW_MCP_COMMAND=openclaw`, `OPENCLAW_MCP_ARGS="mcp serve"` |
| **Auth errors on a custom OpenClaw MCP endpoint** | Verify `OPENCLAW_MCP_AUTH_TOKEN` matches what the server expects; the `Bearer` prefix is stripped automatically |
| **stdio mode: process exits immediately** | Check `OPENCLAW_MCP_COMMAND` path, prefer `openclaw`, and verify `OPENCLAW_MCP_ARGS` starts with `mcp serve` |

---

## Security

- Store `OPENCLAW_MCP_AUTH_TOKEN` in `.env`, not in source code, when you use a custom HTTP/SSE transport.
- Use a read-only token where possible — OpenSRE only reads from OpenClaw, it never writes.
- For hosted instances, ensure the MCP endpoint is behind HTTPS.
